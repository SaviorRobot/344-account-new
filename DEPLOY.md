# 清风公账 - 部署指南

## 方案概述

本项目使用 **Node.js + SQLite + Server-Sent Events**，需部署到支持持久化磁盘的平台。

## 前置准备

### 1. 创建 GitHub 仓库

使用你的 GitHub 账号创建仓库：
- 仓库名称：`344-account`
- 仓库类型：公开（按用户要求）
- 添加 `.gitignore` 以排除数据库文件和敏感配置

### 2. 推送代码

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/your-username/344-account.git
git push -u origin main
```

### 部署步骤（以 Railway 为例）

1. **登录 Railway**：访问 [railway.app](https://railway.app)，使用 GitHub 账号登录
2. **创建新项目**：点击"New Project" → "Deploy from GitHub repo"
3. **连接仓库**：选择你的 `344-account` 仓库
4. **配置环境变量**：
   - `ACCESS_PASSWORD`：访问密码（必填）
   - `NODE_ENV`：`production`
5. **配置持久化**：
   - 在 Railway 中创建新的 Volume
   - 将 Volume 挂载到 `/app/data`
6. **部署**：等待部署完成

## 环境变量配置

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `ACCESS_PASSWORD` | 否 | 空（无密码） | 访问密码，设置后需要登录 |
| `PORT` | 否 | 3000 | 服务端口 |
| `LEDGER_DB_PATH` | 否 | data/ledger.db | 数据库路径 |
| `SESSION_SECRET` | 否 | 自动生成 | 会话密钥 |
| `NODE_ENV` | 否 | development | 运行环境，production 启用安全 Cookie |

## 访问方式

1. 用户打开部署后的网址
2. 如果设置了访问密码，会看到登录页面
3. 输入正确密码后进入账本界面
4. 可以进行增删改查操作，数据实时同步

## 数据备份与恢复

### 备份

1. **手动备份**：下载 `data/ledger.db` 文件
2. **CSV 导出**：在应用中点击"导出"按钮，下载 CSV 文件

### 恢复

1. **数据库恢复**：将备份的 `ledger.db` 文件上传到 `/app/data` 目录
2. **CSV 导入**：需要手动编写导入脚本

## 安全注意事项

1. **访问密码**：务必设置强密码，防止未授权访问
2. **持久化磁盘**：必须配置，否则重启后数据会丢失
3. **代码安全**：虽然仓库公开，但敏感信息（密码、数据库连接）通过环境变量配置，不会暴露在代码中
4. **成员信息**：代码中包含真实成员姓名，请注意隐私保护

## 故障排查

### 数据丢失

原因：未配置持久化磁盘

解决：在部署平台配置 Volume 并挂载到 `/app/data`

### 无法访问

原因：端口未开放或防火墙限制

解决：确保平台开放了配置的端口（通常平台会自动处理）

### 实时同步不生效

原因：SSE 连接被代理或防火墙阻断

解决：检查平台是否支持 WebSocket/SSE，或尝试使用其他部署平台
